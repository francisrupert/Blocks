ESB Component Frame Setup

In order for the Component Frame tool to function Blocks has to be able to find a file called component_frame_template.html, which it will assume is in the same directory as any documents that are using Component Frame 


your_project/
	your_document.html
	component_frame_template.html

A default example of component_frame_template.html is included with the latest release of Blocks

component_frame_template.html

component_frame_template.html can be modified but there are two requirements illustrated in the default version of the file:

  • Line 6: It must include esb.min.css
  • Line 7: It must include esb.min.js

					

				


				
How it works

				

					

						
Component Frame Parsing

						
When you create a Component Frame snippet in your document (your_document.html)...



<!-- your_document.html -->
...
<div data-esb-frame="header" data-esb-variation="base"></div>
...


						

							…Blocks will parse that snippet and create an Frame instance containing an <iframe> element with a src attribute of:
						


component_frame_template.html?data-esb-component=header&data-esb-variation=base

		
						

							When component_frame_template.html loads it will use the query string parameters on the URL to automatically load the requested component into the empty body tag.
						

					

				


				
Modifying component_frame_template.html

				

					

						

							The default component_frame_template.html will only load the component specified via the URL and its corresponding Javascript and CSS. If you have additional styles and Javascript that your components rely on in order to render and function properly, modify component_frame_template.html to include those assets.  
						

					

					



<!DOCTYPE HTML>
<html>
	<head>
		<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
		<title>Component Frame Template</title>
		<link rel="stylesheet" href="/path/to/esb.min.css">
		<link rel="stylesheet" href="/path/to/my_global_styles.css">
		<script src="/path/to/esb.min.js"></script>
		<script src="/path/to/my_global_scripts.js"></script>
	</head>
	<body> 
	</body>
</html>


					

				


				
				
Overriding component_frame_template.html

				

					

						

							If you need to change the location of component_frame_template.html or possibly use multiple template files, you can specify the location of the file using the data-esb-component-frame-template attribute.
						

					

					



<!-- your_document.html -->
...
<div data-esb-frame="header" 
	 data-esb-variation="base"
	 data-esb-component-frame-template="../alternate-directory/component_frame_template.html"
></div>
...
<div data-esb-frame="header" 
	 data-esb-variation="base"
	 data-esb-component-frame-template="alternate_template.html"
></div>
...


					

				


				
Overriding component loading target

				

					

						

							By default the component specified in the query params when component_frame_template.html loads will be injected into the <body> tag of the template. If you create a custom template and want the component to be loaded into a different element on that template, you can specify that behavior with the data-esb-component-frame-template-target attribute.
						

						

							In the example below, the header component will be injected inside <div class="sidebar">
						

					

					



<!-- your_document.html -->
...
<div data-esb-frame="header" 
	 data-esb-variation="base"
	 data-esb-component-frame-template="alternate_template.html"
	 data-esb-component-frame-template-target=".sidebar"
></div>
...





<!-- alternate_template.html -->
<!DOCTYPE HTML>
<html>
	<head>
		<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
		<title>Alternate Component Frame Template</title>
		<link rel="stylesheet" href="/path/to/esb.min.css">
		<link rel="stylesheet" href="/path/to/my_global_styles.css">
		<script src="/path/to/esb.min.js"></script>
		<script src="/path/to/my_global_scripts.js"></script>
	</head>
	<body> 
		<div class="row">
			<div class="main">
				
			</div>
			<div class="sidebar">
				
			</div>
		</div>
	</body>
</html>


					

				


			
Additional Component Attributes

			

				

					
Because Component Frame leverages functionality from Inject, similar options are shared between the two tools.

					
data-esb-place

					

						Inject's data-place option can be leveraged within Component Frame by using the data-esb-place attribute on the Component Frame snippet:
					



<!-- your_document.html -->
...
<div data-esb-frame="header" 
	 data-esb-variation="base"
	 data-esb-place="replace"
></div>
...


					
data-esb-component-source

					

						Inject's data-source option can be leveraged within Component Frame by using the data-esb-component-source attribute on the Component Frame snippet. This should be the source of your components directory relative to the location of your component_frame_template.html.
					



<!-- your_document.html -->
...
<div data-esb-frame="header" 
	 data-esb-variation="base"
	 data-esb-component-source="/path/to/components"
></div>
...
